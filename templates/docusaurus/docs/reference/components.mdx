# Components

All 16 component groups in the kit's
[BEM contract](https://github.com/devcrocod/kotlin-docs-kit/blob/master/docs/contracts/components.md)
work in this starter. The table below maps the **authoring syntax** you
write in MDX to the component you get in the rendered page.

## Parameter table example

```mdx
<Params
  items={[
    { name: 'name', type: 'string', description: "The user's display name." },
    { name: 'id', type: 'string', required: true, description: 'Unique identifier.' },
    { name: 'roles', type: 'string[]', description: 'List of role strings. Defaults to [].' },
  ]}
/>
```

## State

For empty / loading / error placeholders:

```mdx
<State title="No results" icon={<span aria-hidden>📭</span>}>
  Try a different search term, or clear the active filter.
</State>
```

## Named imports (outside MDX)

For TSX / JSX files in `src/pages/` or `src/components/`, import the
components by name — there are no globals outside MDX:

```tsx
import { Callout, Card, CardGrid, Hero, Badge } from '@ktdocs/docusaurus-preset/components';
```

## What you cannot customise

The DOM emitted by these components (class names, nesting, ARIA roles) is
fixed by contract. The visual appearance is fully tweakable through CSS
variables — see the [tokens reference](tokens.mdx).
