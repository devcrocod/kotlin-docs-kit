# Cards & feature grids

`<Card>`, `<CardGrid>`, `<FeatureGrid>` and `<Hero>` are first-class MDX
components — no inline HTML required.

## Auto-fit card grid

`<CardGrid>` is the auto-fit `.kt-card-grid` — column count adapts to the
container width.

  <Card href="../guides/basic-usage" title="Author guide" arrow>

  </Card>
  <Card href="../guides/customization" title="Customise" arrow>
    CSS variable overrides, logo swap, swizzling.
  </Card>
  <Card href="../getting-started/deploy" title="Deploy" arrow>

  </Card>
</CardGrid>

## Fixed three-column FeatureGrid

`<FeatureGrid cols={3}>` pins the column count regardless of viewport
width — useful for landing-page feature sections.

<FeatureGrid cols={3}>
  <Card title="Server">Spring, Ktor, http4k. Native binary, JVM jar, or container.</Card>
  <Card title="Mobile">Compose Multiplatform for iOS + Android from a single codebase.</Card>
  <Card title="Web">Kotlin/JS with React, or Compose for Web.</Card>
</FeatureGrid>

## Source

```mdx
<CardGrid>
  <Card href="/getting-started/installation" title="Getting started" arrow>
    Install and ship your first page.
  </Card>
</CardGrid>

<FeatureGrid cols={3}>
  <Card title="Server">…</Card>
  <Card title="Mobile">…</Card>
  <Card title="Web">…</Card>
</FeatureGrid>

<Hero
  title="Kotlin-styled docs,"
  gradient="powered by Docusaurus."
  tagline="…"
  actions={
    <a className="kt-button kt-button--primary" href="…">
      Get started
    </a>
  }
/>
```

Drop the `href` to render a non-link card. Drop `arrow` to skip the
trailing arrow glyph. `<Card>` becomes `<a>` when `href` is set and `<div>`
otherwise — both render the same BEM block.
