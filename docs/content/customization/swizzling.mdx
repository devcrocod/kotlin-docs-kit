# Swizzling and overrides

Each engine ships its own mechanism for replacing a template or component the kit provides. Use this layer when you need to change **structure**, not just visuals.

## Docusaurus — swizzling

Docusaurus' swizzle CLI handles the boilerplate:

```bash
pnpm docusaurus swizzle @ktdocs/docusaurus-preset Navbar --wrap
```

The `--wrap` form generates a thin wrapper that calls into the kit's swizzled component. Use it whenever you can — the kit's swizzles add the BEM classes that the contract depends on, and ejecting them entirely means re-implementing that part of the visual layer.

### Available swizzles

The preset re-skins these `theme-classic` components:

- `Layout` — host wrapper for `.kt-docs-shell-host`
- `Navbar`, `Footer`
- `DocSidebar`, `TOC`, `DocBreadcrumbs`, `DocPaginator`
- `CodeBlock` (premium codeblock — language label, copy, line highlighting)
- `MDXComponents` (registers `Callout`, `Card`, `Hero`, `CodeTabs`, `Specimen`, etc. as globals)

### `@theme-init` vs. `@theme-original`

When you swizzle a kit component, the generated stub will import from `@theme-original/X`. If the swizzle lives **inside another Docusaurus preset** (not inside a regular site), `@theme-original/X` resolves back into your own preset and crashes the build with a self-import loop. Use `@theme-init/X` instead — that resolves one step earlier in the chain.

Inside a regular site (not a preset), `@theme-original` is fine.

## Hugo — layout lookup

Hugo's layout lookup is simply "your `layouts/` before the theme's". Drop a file at the matching path and it wins:

```
layouts/
  partials/
    footer.html                   # wins over module's footer
  shortcodes/
    myhero.html                   # add a custom shortcode
  _default/
    single.html                   # replace single-page template
```

No CLI ceremony — Hugo just walks the lookup chain.

## What to swizzle, what to override in CSS

| Goal                                                           | Right layer                                                               |
| -------------------------------------------------------------- | ------------------------------------------------------------------------- |
| Tweak colors / fonts                                           | [CSS variable override](./css-overrides.mdx)                              |
| Add a footer column or change the copyright text               | Engine config (`extra`, `themeConfig.footer`, `params`)                   |
| Replace the navbar entirely with custom HTML                   | Override / swizzle                                                        |
| Inject analytics, OpenGraph tags, or other `<head>` content    | Override the head partial (Hugo) or swizzle `Root` (Docusaurus)           |
| Change the structure of a component (callout, codeblock, card) | **Don't** — open an issue. The contract is shared with the other engines. |
