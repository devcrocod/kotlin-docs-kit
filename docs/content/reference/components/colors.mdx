# Colors

Five colour families: the brand palette, supporting accents, neutrals, semantic state colours, and the dedicated code-block surfaces.

## Brand — Purple

The signature Kotlin purple ramp (50–900). Use `500` for solid brand surfaces, `100` for soft backgrounds, `700+` for text on light surfaces.

<Specimen src="colors-purple" height={340} title="Kotlin purple ramp" />

## Accents

Supporting brand accents — magenta, red, pink, orange, blue. Use these for variation in hero backgrounds, badge variants, and accent text. They share the same `50/100/200/300/400/500/600` step structure as purple.

<Specimen src="colors-accents" height={440} title="Brand accent ramps" />

## Neutrals

14-step gray ramp from pure white (`gray-0`) to near-black (`gray-950`). The light theme composes surfaces from `gray-0..200`; the dark theme uses `gray-800..950`.

<Specimen src="colors-neutrals" height={320} title="Neutral ramp" />

## Semantic

Status hues for callouts, badges, and inline state — `green` (success), `amber` (warning), `red` (danger), `cyan` (info). Each has a `100/300/500/600` ramp.

<Specimen src="colors-semantic" height={340} title="Semantic palette" />

## Code

Dedicated code-block surfaces (`--code-bg`, `--code-fg`, `--code-line-bg`, `--code-line-hl`) plus 8 Pygments-style token colours (`--code-keyword`, `--code-string`, `--code-number`, `--code-comment`, `--code-function`, `--code-type`, `--code-punctuation`, `--code-tag`).
