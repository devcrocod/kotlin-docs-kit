# Components

The 16-group baseline. Every specimen is a static HTML reference; the same DOM is emitted by each engine implementation. For the full contract (class names, ARIA roles, what is and isn't customizable), see [`docs/contracts/components.md`](https://github.com/devcrocod/kotlin-docs-kit/blob/master/docs/contracts/components.md).

## Targets

Kotlin Multiplatform target badges for API reference pages — a labelled row of pills (`.kt-targets`). Drop `<Targets>` at the top of a symbol's page to show where it is available. Registered globally, so no import is needed in MDX.

<Targets targets={['jvm', 'android', 'ios', 'js', 'wasmJs', 'native']} />

```mdx
<Targets targets={['jvm', 'android', 'ios']} />
```

Supported target keys: `jvm`, `js`, `wasmJs`, `native`, `android`, `ios`, `macos`, `watchos`, `tvos`, `linux`, `mingw`. Pass `title={null}` to drop the leading label.

To flag API status, pair it with the `deprecated` and `experimental` admonitions the preset registers:

:::experimental
This symbol is experimental and may change in a future release.
:::

```mdx
:::deprecated
Use `newApi()` instead — this will be removed in 2.0.
:::
```
